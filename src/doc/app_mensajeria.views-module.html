Funcion: Panel principal de administracion de mensajes

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
Returns:
Si el usuario se encuentra logueado retorna un objeto HttpResponse del template mensajes.html renderizado con el contexto {'usuario': u, 'lista': lista}. Sino, retorna un objeto HttpResponseRedirect hacia '/login'.

nuevoMensaje(request)

source code 

Funcion: Se ocupa de crear un nuevo mensaje

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
  • id_sender - Identificador del usuario remitente.
  • id_receiver - Identificador del usuario destinatario.
Returns:
Si el usuario no esta logueado retorna un objeto HttpResponseRedirect hacia '/login'. Si el metodo no es POST retorna un objeto HttpResponseRedirect hacia el indice de mensajes.

eliminarMensaje(request, id_mensaje)

source code 

Funcion: Se ocupa de eliminar un mensaje

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
  • id - id del mensaje a ser eliminado.
Returns:
Si el usuario se encuentra logueado y el usuario dado es eliminado exitosamente retorna un objeto HttpResponseRedirect hacia el indice de mensajes. Sino, retorna un objeto HttpResponseRedirect hacia '/login'.

verMensaje(request, id_mensaje)

source code 

Funcion: Se ocupa de visualizar el contenido de los mensajes

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
  • id_mensaje - Identificador del mensaje que se visualizara.
Returns:
Si el usuario esta logueado, se retorna una objeto HttpResponse del template ver-mensaje.html con el contexto {'usuario':u, 'mensaje':mensaje}. Sino, retorna un objeto HttpResponseRedirect hacia /login.
