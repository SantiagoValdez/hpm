indexFase(request, id_proyecto)

source code 

Funcion: Panel principal de administracion de las fases del proyecto

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
  • id_proyecto - Identificador del proyecto del cual se visualizan sus fases.
Returns:
Si el usuario se encuentra logueado retorna un objeto HttpResponse del template fases.html renderizado con el contexto {'usuario': u, 'proyecto' : proyecto, 'lista': lista}. Sino, retorna un objeto HttpResponseRedirect hacia '/login'.

nuevaFase(request, id_proyecto)

source code 

Funcion: Se ocupa de crear una nueva fase

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
  • id_proyecto - Identificador del proyecto al cual se le agregara una nueva fase.
Returns:
Si el usuario se encuentra logueado y si una nueva fase es creada exitosamente retorna un objeto HttpResponse del template fases.html renderizado con el contexto {'usuario' : u, 'proyecto' : proyecto, 'fases' : fases, 'mensaje' : 'Se creo la fase con exito'}. Sino, retorna un objeto HttpResponseRedirect hacia '/login'.

eliminarFase(request, id_proyecto, id_fase)

source code 

Funcion: Se ocupa de eliminar una fase del proyecto

Parameters:
  • request - Objeto que se encarga de manejar las peticiones http.
  • id_proyecto - Identificador del proyecto del cual se elimina un fase. Utilizado para renderizar el indice de fases.
  • id_fase - Identificador de la fase a ser eliminada.
Returns:
Si el usuario se encuentra logueado y el proyecto es eliminado exitosamente retorna un objeto HttpResponseRedirect hacia el indice de fases del correspondiente proyecto. Sino, retorna un objeto HttpResponseRedirect hacia '/login'.
